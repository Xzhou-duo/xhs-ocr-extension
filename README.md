# 小红书图片文字提取插件

一款用于 Chrome / Edge 的 Manifest V3 浏览器扩展。在小红书网页版图文笔记中选择帖子正文图片，调用百度 OCR 提取文字，并支持复制结果、历史记录和重复提取拦截。

当前版本：`1.0.1`

## 主要功能

- 自动采集小红书图文笔记中的正文图片。
- 过滤作者头像、评论头像、图标等非正文图片。
- 支持单图和多图笔记，并保持图片原有顺序。
- 多图选择器支持左右翻页、鼠标滚轮、触控板、鼠标拖动、触屏滑动和键盘方向键。
- 可自由选择单张、多张或全部图片进行 OCR。
- 使用百度通用文字识别接口串行处理图片。
- 支持复制单张结果或一次复制全部结果。
- 自动保存最近的提取记录，并按帖子分组查看。
- 已提取图片优先复用历史结果，避免重复消耗 OCR API 额度。
- 支持对单张历史结果主动执行“重新识别”。
- 支持 Token 缓存、失败重试和中文错误提示。

## 隐私与数据说明

- 百度 OCR 的 `API Key`、`Secret Key`、Token 和提取历史保存在浏览器本机的 `chrome.storage.local`。
- 扩展不会把密钥或提取内容发送到扩展作者的服务器。
- 图片数据仅在执行识别时发送至百度 OCR 接口。
- 小红书图片由扩展 Service Worker 获取并转换为 Base64，避免页面跨域限制。
- 清除浏览器扩展数据或卸载扩展后，本地设置和历史记录可能丢失。

## 使用前准备

需要拥有百度智能云文字识别服务的 API 凭证：

1. 登录百度智能云控制台。
2. 创建文字识别应用。
3. 获取应用的 `API Key` 和 `Secret Key`。
4. 确认账户具有通用文字识别接口的可用额度。

百度服务的开通方式、免费额度和计费规则可能调整，请以百度智能云当前说明为准。

## 安装扩展

### 通过源码加载

1. 下载或克隆本仓库。
2. 打开 Chrome 的 `chrome://extensions`，或 Edge 的 `edge://extensions`。
3. 开启右上角的“开发者模式”。
4. 点击“加载已解压的扩展程序”。
5. 选择仓库中的 `xhs-ocr-extension` 目录。

修改扩展代码后，需要在扩展管理页点击“重新加载”，并刷新已打开的小红书页面。

### 通过 ZIP 安装开发版本

项目中的发布 ZIP 解压后，选择其包含 `manifest.json` 的根目录，通过“加载已解压的扩展程序”安装。

## 配置百度 OCR

1. 点击浏览器工具栏中的扩展图标。
2. 输入百度 OCR 的 `API Key` 和 `Secret Key`。
3. 点击保存。
4. 刷新小红书页面，使最新配置和脚本生效。

更换 API 凭证后，扩展会废弃旧 Token 缓存并重新获取授权。

## 提取帖子文字

1. 在小红书网页版打开一篇图文笔记详情。
2. 点击页面右下角的“提取文字”按钮。
3. 展开“选择图片”区域。
4. 勾选需要识别的图片，也可以使用“全选”。
5. 点击“提取所选”或“提取全部”。
6. 等待识别完成后，复制单张结果或点击“复制全部”。

图片会串行识别，以降低接口限流和瞬时请求失败的概率。

## 浏览和选择多张图片

当帖子图片超过当前可见范围时，可以通过以下方式浏览后面的图片：

- 点击图片区域左右两侧的翻页箭头。
- 将鼠标放在图片区域内并滚动滚轮。
- 使用触控板横向或纵向手势。
- 按住鼠标左键拖动图片列表。
- 在触屏设备上直接横向滑动。
- 聚焦图片列表后按键盘 `←` 或 `→`。

翻页箭头每次移动约四张图片。到达列表开头或末尾时，对应方向的箭头会自动隐藏。滚动不会改变图片的勾选状态。

## 重复提取与 API 额度保护

扩展会为成功识别的图片保存历史结果：

- 再次提取同一图片时，直接展示历史结果，不调用百度 OCR。
- 刷新页面导致图片 CDN 地址变化时，会使用“帖子 ID + 图片序号”匹配历史。
- 部分图片已有历史时，仅对新增图片调用 OCR。
- 所选图片全部已有历史时，会明确提示本次不会重复调用 API。
- 只有用户主动点击历史结果旁的“重新识别”，才会再次调用 OCR 并覆盖该图片的历史结果。

“类似内容”指扩展能够通过图片地址或同一帖子中的图片序号确认是同一张图片，不包含图片视觉相似度分析。

## 历史提取记录

点击面板顶部的历史图标可以查看历史记录：

- 历史记录按帖子分组，并按最近更新时间排序。
- 帖子列表支持删除单篇帖子的全部历史。
- 帖子详情支持查看图片、复制单张文字和复制全部文字。
- 全局“清空记录”只在帖子列表页显示，避免与删除当前帖子混淆。
- 清空记录会删除扩展本地保存的全部提取历史，操作前会再次确认。

## 图片采集规则

扩展优先从小红书帖子轮播或媒体区域中按顺序采集图片：

- 单图帖子可以直接采集，不会为了凑够多图而扫描整个详情页。
- 排除类名包含头像、作者、用户信息或评论特征的图片。
- 兜底扫描会过滤小尺寸头像和图标。
- 去重时忽略图片 URL 的临时签名和格式参数，但实际下载仍使用完整 URL。
- 对懒加载图片设置 `loading="eager"`，并在请求前重新读取当前图片地址。

小红书调整页面 DOM 后，图片选择器可能需要同步更新。

## 错误处理

扩展会区分并提示常见问题：

- API Key 或 Secret Key 无效。
- 百度 Token 过期。
- OCR 请求过于频繁。
- OCR 调用额度不足。
- 图片格式或大小不符合要求。
- 图片获取失败。
- 网络连接失败或请求超时。

普通失败会等待后自动重试一次。Token 失效时会强制刷新 Token 后重试。单张图片失败不会清空其他已成功的结果。

## 项目结构

```text
.
├── README.md
├── HANDOFF.md
├── package.json
├── tests/
│   └── core_utils.test.js
└── xhs-ocr-extension/
    ├── manifest.json
    ├── core_utils.js
    ├── background.js
    ├── content_script.js
    ├── content_style.css
    ├── popup.html
    ├── popup.js
    └── icons/
```

核心模块：

- `content_script.js`：页面按钮、图片采集、图片选择器、OCR 流程和历史界面。
- `background.js`：百度 Token 缓存、跨域图片获取和 Base64 转换。
- `core_utils.js`：图片去重、历史匹配、滚动状态和错误映射等共享逻辑。
- `popup.js`：百度 OCR 凭证的读取与保存。

## 本地开发

需要安装 Node.js。项目没有运行时 npm 依赖，测试使用 Node.js 内置测试运行器。

```powershell
npm.cmd run check
npm.cmd test
```

- `npm.cmd run check`：检查扩展 JavaScript 文件语法。
- `npm.cmd test`：运行 URL 去重、历史复用、图片尺寸过滤、滚动状态、Token 和错误映射测试。

## 手工回归建议

每次修改图片采集或界面交互后，建议验证：

1. 单图帖子只出现正文图片，不出现作者头像。
2. 多图帖子能按原顺序采集全部图片。
3. 18 张图片可以浏览并选择到最后一张。
4. 左右箭头、滚轮、拖动和键盘方向键均可滚动。
5. 图片选择状态和数量统计在滚动后保持正确。
6. 已提取帖子刷新后能复用历史结果，不重复调用 API。
7. “重新识别”仍能主动更新单张结果。
8. 历史帖子详情不显示全局“清空记录”按钮。

## 已知限制

- 目前仅支持小红书网页版图文笔记，不处理视频内容。
- 图片识别质量取决于原图清晰度和百度 OCR 服务。
- 小红书 DOM、轮播结构或图片地址规则变化时，采集逻辑可能需要更新。
- 正式发布至浏览器商店前，还需要准备隐私政策、商店介绍和正式宣传素材。
